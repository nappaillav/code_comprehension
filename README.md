# Code Comprehension Examples

This repository contains examples for improving code comprehension by providing step-by-step reasoning in both the forward and backward directions.

## Purpose

The goal is to teach a model how to reason about code by showing:

* **Forward reasoning** – starting from the given input (e.g., code, function, or repository context) and reasoning step by step to reach the expected output.
* **Backward reasoning** – starting from the expected output and reasoning backwards to identify the code, logic, or evidence that produced it.

Each example may also include positive and negative feedback to demonstrate good and poor reasoning patterns.

## Structure

Each example consists of:

* **Input** – Code, repository context, or task description.
* **Output** – Expected answer or result.
* **Forward Reasoning** – Step-by-step reasoning from input to output.
* **Backward Reasoning** – Step-by-step reasoning from output back to the supporting code or evidence.
* **Positive Feedback** – Highlights what makes the reasoning correct.
* **Negative Feedback** – Points out incorrect assumptions, missing evidence, or reasoning mistakes.

## Goal

By exposing the model to both forward and backward reasoning, along with examples of good and bad reasoning, the dataset aims to improve:

* Code understanding
* Multi-step reasoning
* Evidence-based explanations
* Overall reasoning quality

This repository is intended as a collection of reasoning examples that can be used for training, fine-tuning, or evaluating code comprehension models.
